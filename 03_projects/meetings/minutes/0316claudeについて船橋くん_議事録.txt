## 議題・主要な議論

### 1. Claude Code の概要と環境構築
- 船橋くんが「ChatGPT・GeminiよりもClaude推奨」とアドバイスしたことが水谷さんのClaude移行のきっかけ
- ターミナル単体での操作は視認性が悪く非推奨。**Cursor / Antigravity（Windsurf）/ VSCode のいずれかと組み合わせること**を推奨
- IDE を使うメリット：①ファイルエクスプローラーで参照範囲が視覚的にわかる、②PDFやCSVをプレビュー可能、③参照範囲を絞ることでセキュリティも向上
- 水谷さんの現状：インストール済み・ターミナルで動かしている・Cursor も入っているが画面上部の表示方法がわからず止まっている
→ **Cursor の「Open Folder」からプロジェクトフォルダを開く方法をその場でデモ**

### 2. クロン実行（Cron）とパイプライン自動化
- **クロン実行**＝指定時刻・間隔でスクリプトを自動実行する仕組み（Linuxの古典機能）
- Claude Code に「このパイプラインをクロン実行して」と自然言語で指示するだけで自動化が組める
- 例：Google Meet録画が終わったら → ローカルに保存 → 文字起こし → 議事録＋提案書作成 → Google Drive に保存
- 難易度：パイプライン設計自体は比較的容易。クロンの完全自動安定動作にはトライ＆エラーが必要（PCの電源・権限等のトラブルが起きやすい）
- PC が常時起動しているかどうかが前提条件になる

### 3. 名刺スキャン → 顧客管理 → 提案書自動生成パイプライン（船橋くんの実例）
- スマホで名刺を撮影 → Google Drive にアップ → 24時間稼働デスクトップPCがローカルOCRで読み取り → 会社リサーチ → 仮説3件を生成 → 会社テンプレートPPTX（スキル使用）で提案書作成 → Google Drive に自動保存
- 商談30分後（場合によっては商談中）に提案書が完成する状態を実現済み
- 社員は「名刺を写真撮ってドライブに入れる」だけでよい
- SalesForce 等の高コストSaaSをGoogle Workspace + Claude Codeで代替可能という認識を共有

### 4. PPTX スキル・音声録音活用
- Anthropic 公式の PPTX スキルで提案書を自動生成（会社テンプレートを事前に登録して微調整）
- 商談の音声録音 → 文字起こし → そのまま課題が提案書に反映される流れが精度高く機能している
- 画像・ロゴのサイズ調整など細かい修正はスクショ＋言葉で説明して反復修正

### 5. 日次タスク管理の自動化（水谷さんの課題）
- Slack・メール・コミュニケーションツールのメンション・タスクを朝一で一覧化したい
- クロン実行でSlack・Googleカレンダー・Gmailを繋いで朝のレポートを自動生成することは可能
- LINEは公式LINEであればAPI連携可能。個人LINEは難しい

### 6. 動画生成パイプライン（水谷さんの興味）
- シーン生成（スタートフレーム作成→動画化）の自動化は技術的には可能
- **課題：動画生成APIのコストが非常に高い**（個人・小規模制作には負担が大きい）
- 完成通知をスマホに届けて確認するフローは設計可能

### 7. 非エンジニアの学習アプローチ
- 「走りながら覚える」＝やりたいことに対して必要な知識をその都度インプットするのが最もコスパが良い
- わからない単語はClaude に短く解説させればよい

### 8. サイバーとの取引可能性
- 水谷さんより「サイバーとの取引実績を残したい」という意向を共有
- 船橋くんより「正式なフローで、お互いに価値があるなら全然いいと思う」との回答

## 決定事項・合意内容
- Claude Code は Cursor を使って環境を整備する
- デスクトップに作業フォルダを作成し、Cursor の Open Folder で開く
- 既存のMCPサーバー設定（Google Drive・Gmail等）はそのまま流用可能
- パイプラインの完全クロン化が難しい場合は「毎回手動で全部実行して」と Claude Code に一括依頼する方法で代替可能

## 事業への示唆

## Claude Code活用ミーティング（船橋くん）からの事業示唆

### ■ 機会

- **営業DX即戦力**：名刺スキャン→会社リサーチ→提案書自動生成パイプラインは、化粧品のB2B卸営業（小売バイヤー・セレクトショップ等へのアプローチ）に直接応用できる。商談の熱量が冷めないうちに提案書を出せることは競合他社との明確な差別化になる
- **SaaSコスト削減**：SalesForce・HubSpot等のCRM費用をGoogle Workspace + Claude Codeで代替できる可能性。月額数万円規模のコスト削減につながりうる
- **音声→コンテンツ自動化**：商談録音→議事録→社内共有のフローは、既に本システムで稼働中。これをEC商品企画MTG・ブランドブリーフィング等にも展開できる
- **マーケ定期タスクの自動化**：SNS投稿スケジュール確認・EC売上レポート取得・競合価格チェック等の定型業務をクロン実行で自動化できれば、マーケ担当の工数を戦略業務にシフトできる
- **動画コンテンツ量産（中期）**：APIコストが下がれば、商品紹介動画・SNS用ショートコンテンツの自動生成パイプラインが現実的になる。今から設計だけ進めておく価値がある
- **社員への展開容易性**：「名刺を写真撮ってドライブに入れるだけ」のように、フロントのUXをシンプルにする設計ができれば、ITリテラシーが低いスタッフでも活用できる

### ■ リスク・注意点

- **クロン実行の安定性**：PC常時起動が前提条件。モバイルワークや出張時には自動化が止まるリスクがある。Mac mini等の専用マシンを検討すべきタイミングが来る可能性
- **動画生成API費用**：現時点では個人・小規模では割高。EC商品の動画量産をAIに任せる場合は費用対効果の試算が必要
- **トライ＆エラーコスト**：クロン実行まで含めた完全自動化には相応のデバッグ時間が必要。「即日完全自動化」という過度な期待を持たず、段階的に育てる計画が必要
- **MCPサーバー・API依存**：外部APIの仕様変更・認証切れで突然パイプラインが止まるリスクがある。重要フローには手動バックアップ手順を持っておくべき
- **機密情報の取り扱い**：顧客名刺情報・商談録音を自動処理するパイプラインを組む場合、データの保存先・アクセス権限・個人情報保護法への対応を事前に整理する必要がある

### ■ 戦略的示唆

- 船橋くんのように「Claude Codeを使いこなせる人材」との継続的な関係構築は費用対効果が非常に高い。正式な業務委託または顧問契約を検討する価値がある
- 現在進行中のEC売上強化SNS施策（水谷・3/16戦略会議方針）においても、コンテンツ生成・分析レポートの自動化を組み込むことで施策スピードを上げられる可能性がある
- 自社で構築したClaude Codeパイプラインを「資産」として整備していくことが、中長期的な競争優位の源泉になりうる
