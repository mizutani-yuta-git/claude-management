## 課題（水谷さんの実感 + 資料分析）

### 1. カンプ精度と判断基準の不一致
- AIカンプの商品部分は仮生成物なので、方向性の良し悪しを判断できない状態でFBKが行われた
- 「背景・世界観」と「商品クオリティ」が混在した状態でカンプ確認していた
- 撮影してみて初めて判断できる、という後追いになりやすい構造

### 2. AIゆえに「決め」がなくなる
- AIは変更コストが低いため、CDやADによる「ここで決める」が発生しにくい
- 従来の撮影フローは「撮影日＝コスト発生の壁」でFIXポイントが自然発生していた
- AIはその壁がないため、意図的に設計しないと際限なく変更が続く

### 3. ブランド表現の「型」が最初にできているか不明
- オリエン〜コアビジュアル生成の段階で、N organicのブランドトーンをAIに落とし込む初期設定（プロンプト・世界観定義）が担保されているか、外から見えない
- 「ブランドのAI翻訳」プロセスが不可視

### 5. AIの偶発性をコントロールできないと活用場面がない
- AIは確率的な生成のため「偶然いいものが出る」ことがある。それを良しとしてしまうと再現性がない
- **人がコントロールするのと同じようにトンマナを統一できる精度**に達していないと、ブランドクリエイティブとしては使えない
- 「何度生成しても同じ世界観に収束する」状態がゴール
- 現状はまだ「出てきたものを選ぶ」フローになっており、「意図したものを出す」フローになっていない
- これが解決されないと活用場面が限定される（バリエーション量産よりも偶発的な1点ものになってしまう）

### 4. プロダクション側のブランド・事業理解が不足している可能性
- AI表現の技術力の問題だけでなく、**パートナーとしてのブランド理解・事業理解の深度が足りていないかも**
- そのための「時間」が制作プロセスに組み込まれていない
- 従来の撮影プロダクションにも同様の課題はあるが、AIは「とりあえず生成できてしまう」ため理解不足が成果物に出やすい
- ブリーフ＋キックオフだけでは不十分で、ブランドの歴史・顧客・NGの文脈を共有するセッションが必要かもしれない

## 改善ワークフロー案（ドラフト）

```
Phase 0：ブランドAI設定の合意（新設）
  amanaがN organicのスタイルプロンプト・世界観定義を作成
  → シロクがレビューし合意（ブランド担保の起点）

Phase 1：世界観カンプ確認
  判断軸を分割して評価
  ① 背景・世界観・ライティング方向 → YESなら次へ
  ② 商品表現は「ダミー画像として見る」と明示（判断対象外）

Phase 2：商品合成確認（新設）
  Phase 1承認後、実商品写真 or 商品モックをカンプに合成
  → レイアウト・トリミング・商品見え方を確認
  → FIXしてから撮影へ

Phase 3：撮影
  Phase 2のカンプを「設計図」として使用

Phase 4：Multisource展開
```

### FIXゲート設計

| ゲート | タイミング | 決めること |
|--------|-----------|-----------|
| G1 | Phase 0完了時 | ブランドのAI表現定義をLock |
| G2 | Phase 1完了時 | 世界観・方向性をLock |
| G3 | Phase 2完了時 | レイアウト・商品見えをLock → 撮影GO |

## チャネル別ニーズ・制約メモ

### PF（amazon・楽天 等 外部モール）
- **商品を大きく見せるトリミング**が前提（ECモールは商品主役）
- **セット組バリエーション**を多数作りたい（例：LM＝ローション＋ミルクのセット画像）
- **モール独自のアイコン・訴求テキストを重ねる**運用が多い
  → 余白・背景の設計が重要。テキストレイヤーを重ねやすいシンプルな背景が◎
  → コアビジュアルとは別に「PF専用レイアウト」の設計が必要かもしれない

**I2Mワークフローへの示唆**
- Multisourceの段階でPF用トリミング・背景シンプル版を派生させる設計にする
- セット組は商品パターンだけ変えればよい構造なのでAIとの相性が高い用途
